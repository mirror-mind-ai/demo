# Processo

O método pelo qual o Conjunto é construído. Captura a disciplina
herdada de XP e adaptada para um construtor solo trabalhando com
Mirror Mind: design antes de código, histórias pequenas com validação
imediata, refatoração no ciclo, testes verdes em todo commit,
documentação viva.

## Documentos

- **[Development guide](development-guide.md)** — o manual de como o
  trabalho acontece: princípios, hierarquia do roadmap (CV, Epic,
  Story), ciclo completo de uma story (Plan, Implement, Test,
  Document, Review, Status, Push), e convenções de Git.
- **[Worklog](worklog.md)** — log do trabalho realizado por dia. Cada
  entrada é uma observação curta sobre o que foi feito, o que foi
  aprendido, o que ficou para depois.
