# v0.2.1 — Last seen

O Conjunto ganha o primeiro sinal de presença: ao abrir o perfil de
outro membro, vejo quando ele esteve online por último. Na listagem
de membros, o mesmo indicador aparece abaixo do papel e empresa.

Não é chat. Não é status de "online/offline". É uma pista editorial
de que o grupo respira, de que as pessoas estão passando por aqui.
Coerente com o princípio de engajamento sem vício: o sinal ajuda a
sentir a comunidade viva, sem criar urgência ou competição.

## O que mudou

**Schema.** Coluna `last_seen_at TEXT` na tabela `members`. Migration
idempotente para bancos existentes (mesmo padrão de `currently` e
`avatar_img`). Sem default: membro que nunca acessou tem NULL, e a
UI não mostra nada.

**Middleware.** O `ensureMember` agora faz um UPDATE de
`last_seen_at` a cada request autenticado. Um write por request é
irrelevante em SQLite WAL para sete membros.

**Formatação.** Função pura `formatRelativeTime` em
`src/lib/time.ts`. Recebe ISO string, devolve texto em português:
"agora mesmo", "há 3 horas", "há 2 semanas", "há mais de um mês".
Testável com datas fabricadas, sem banco.

**Perfil.** Abaixo de "No Conjunto desde...", aparece "Última vez
online: há X". Só para outros membros; o próprio perfil omite a
linha.

**Listagem.** Na página `/members`, cada membro mostra o indicador
abaixo do papel. Mesma regra: não aparece para o próprio membro.

## Números

- 1 coluna nova no schema
