# v0.1.0 — Tracer Bullet

*12 de maio de 2026*

**TL;DR**

- **CV0.E1 — Tracer Bullet** completo. Seis stories. O Conjunto existe
  como produto rodável.
- **Stack escolhido boring por design.** TypeScript + Hono +
  better-sqlite3 + HTML server-rendered. Sem Next.js, sem tRPC, sem
  Prisma. Justificativa registrada em [briefing → D1](../project/briefing.md#d1-boring-stack-first).
- **Documentação dentro do produto.** A rota `/docs` lê e renderiza
  `docs/` ao vivo. Edição em markdown aparece imediatamente. ~80
  linhas de código.
- **Auth simulada.** Cookie com id do membro. Troca aberta na home.
  Aceitável em comunidade fechada de sete pessoas que se conhecem;
  intolerável quando abrir externamente. Auth real entra em v0.2.0.
- **Sete fundadores cadastrados via seed.** Os sete do Sustentação:
  Lucas, André, Beatriz, Caio, Diana, Eduardo, Felipe, Gabriela.
- **Cinco fios iniciais.** Conversas reais sobre temas concretos:
  promoção adiada, delegação, saída de empresa, imposter syndrome,
  reorg de squads.

## De onde estávamos

Cinco meses de rascunho conceitual. Eu sabia o produto que queria
construir mas adiava o primeiro deploy porque cada vez que sentava
para começar, a tentação era "fazer com Next.js para já ficar pronto
para v0.3.0".

Em abril, escrevi o briefing. Tomei a decisão D1 (boring stack first).
Em três semanas o produto estava rodando, com docs navegáveis dentro
dele, com os fundadores cadastrados, com fios reais sendo conversados.

O Next.js que eu queria construir há cinco meses ainda não existiria
hoje.

## O que foi feito

**S1 — Tracer bullet servidor + health.** Hono sobre `@hono/node-server`,
porta 3030, rota `/health` que responde "ok". Quinze minutos de
trabalho, mas marco crítico: a partir desse commit, existe um produto.

**S2 — Membros listados e perfis lidos.** Tabela `members`, rotas
`/members` e `/members/:id`. Sem cadastro pela UI ainda; o seed é a
verdade do MVP (decisão D5).

**S3 — Auth simulada por cookie.** Cookie `conjunto_member` guarda o
id do membro ativo. Middleware `ensureMember` garante que sempre há
um membro selecionado. Troca aberta via dropdown na home. Decisão D3
documenta o trade-off.

**S4 — Fios e mensagens persistentes.** Tabelas `threads` e `messages`,
rotas `/threads` e `/threads/:id`. Sem markdown ainda (decisão
consciente de adiamento), sem edição (entra em v0.2.x).

**S5 — Documentação dentro do produto.** Rota `/docs` que lê a árvore
do filesystem em runtime, renderiza markdown com `marked`, reescreve
links relativos para rotas internas. Sidebar de navegação gerada da
estrutura de pastas. Decisão D4 documenta as alternativas avaliadas.

**S6 — Seed reprodutível dos fundadores.** `npm run seed` apaga as
três tabelas e recria do zero. Idempotente. Os fundadores e os fios
iniciais entram nesse script, não em forms.

## O que está consciente fora deste lançamento

- Auth real (magic link via email) — v0.2.0
- Pagamento (Stripe) — v0.2.0
- Markdown nas mensagens de fios — quando alguém pedir explicitamente
- Edição de perfil pelo membro — quando houver mais que sete membros
- Busca na biblioteca — quando o volume justificar
- Notificações por email — quando o ritmo exigir

Lista completa em [briefing → Conscientemente fora do MVP](../project/briefing.md#conscientemente-fora-do-mvp).

## O que aprendi

**Documentar antes de codar não é cerimônia.** O briefing demorou três
dias para escrever. Cada vez que eu ia escrever, percebia uma decisão
que eu ainda não tinha tomado de fato. Escrever o briefing É tomar as
decisões. Quando comecei a codar, não tinha mais decisões pendentes;
era só execução.

**Resistência ao stack sofisticado é difícil.** Eu sabia
intelectualmente que Next.js + tRPC + Prisma era over-engineering para
o MVP. Sentia uma inquietação cada vez que ia escrever HTML
server-rendered. A inquietação é o sinal de que estou seguindo o
costume da indústria, não a régua do problema.

**Documentação dentro do produto vira parte do produto.** Esperava
construir `/docs` como conveniência minha. Acabou virando uma das
páginas que faz mais sentido para um membro que está conhecendo a
comunidade. Onde estamos, como pensamos, o que está em curso, tudo
acessível a um clique.
